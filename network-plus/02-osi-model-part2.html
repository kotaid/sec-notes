L1Physical transmission — cables, UTP, signals, binary
L2Local delivery — NICs, MAC addresses, Ethernet frames, switches

Data Link Sublayers (L2 Revisited)

Before going up the stack, L2 is split into two sublayers:

SublayerNameRole
LLCLogical Link ControlThe NIC communicates with the OS via drivers — provides multiple network protocol support and flow control
MACMedia Access ControlCreates the MAC address, attaches it to the frame with the receiver address, checks FCS, ensures frames travel correctly

Why MACs Aren't Enough

MACs work for small local networks. As networks grow larger, broadcasting MAC addresses floods and blocks the entire network. We need logical addressing — which ignores hardware and lets us break networks into subnets.

The most famous implementation: TCP/IP — Transmission Control Protocol / Internet Protocol.

L3 — Network Layer

// IP Addressing

IP ensures packets reach their destination by assigning a unique IP address to every device. Unlike MACs, IP addresses are stored in software — not hardware.

192 . 168 . 4 . 232
 8bit  8bit  8bit  8bit  =  32 bits total

// Routing

Each router receives a frame, strips it, reads the packet to find the next destination, creates a new frame, and forwards it. This repeats until data arrives at the destination subnet.

// Frames vs Packets

[ FRAME — L2 ]
  └── Destination MAC | Source MAC | Type | [ PACKET — L3 ] | FCS
                                              └── IP Header | Data
// SubnettingIP addresses are divided into subnets to break large networks into manageable, secure segments. Dedicated subnetting writeup coming soon.

L4 — Transport Layer

// Segmentation

Data is often larger than a single frame can carry (Ethernet max: 1500 bytes). L4 chops it into segments and the receiving side reassembles them in order, requesting retransmission of anything missing.

// TCP vs UDP

TCP
Connection-Oriented
  • Verifies connection before sending
  • Guarantees delivery and ordering
  • Used for: web, email, file transfers
  • Slower but reliable
UDP
Connectionless
  • Fires without verifying target is ready
  • No delivery guarantee
  • Used for: VoIP, video, DNS
  • Faster but unreliable

// Ports

A port is a logical number between 1 and 65,536 telling the OS which application should receive incoming data. IP gets you to the building — the port tells you which apartment.

0 – 1,023Well-known — HTTP: 80, HTTPS: 443, SSH: 22, DNS: 53
1,024 – 49,151Registered — assigned to specific applications
49,152 – 65,535Dynamic / ephemeral — temporary, assigned per session

L5 — Session Layer

Manages communication sessions between applications on different systems — initiates, accepts, and closes sessions.

// SOC TipRun netstat -a on Windows to list all existing sessions — useful for spotting unexpected connections during an investigation.

L6 — Presentation Layer

Handles data translation — encoding, decoding, and encryption/decryption between the network and applications. One of the blurriest layers in practice, with many functions overlapping L7.

L7 — Application Layer

Not the apps themselves — the OS built-in APIs that make applications network-aware. Provides encryption, authentication, and data presentation tools.

Encapsulation & Decapsulation

As data passes down the stack, each layer wraps it with its own header. The receiver strips those headers going back up — each layer only reads its own header.

// The Elegant PartEvery layer only reads its own header and passes the rest up — this is what makes OSI truly modular.

SOC Analyst Relevance

IP Spoofing
L3 — Network

Attackers forge source IPs to disguise origin or bypass filters.

Port Scanning
L4 — Transport

Tools like Nmap discover open services. Unusual port activity signals reconnaissance.

TCP SYN Flood
L4 — Transport

Massive SYN requests without completing handshakes exhaust server resources.

Session Hijacking
L5 — Session

Attacker takes over an authenticated session mid-communication.

App Layer Attacks
L7 — Application

SQL injection, XSS, API abuse. Your dev background gives you a serious edge here.

UDP Floods
L4 — Transport

UDP's connectionless nature makes it easy to flood targets with no built-in protection.

// Key Takeaways

MACs work locally — IP addresses enable communication across large networks
Routers strip and rebuild frames at each hop — they only care about IP addresses
TCP is reliable and connection-oriented; UDP is fast and connectionless
Ports tell the OS which application should receive incoming data (1–65,536)
Encapsulation wraps data going down the stack; decapsulation unwraps it going up
Most real-world attacks target L3 and above — understanding the full stack is fundamental
