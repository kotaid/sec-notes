A Brief History of Ethernet

Ethernet was developed by Xerox, then handed over to DIX (Digital Equipment Corporation / Intel / Xerox), who passed it to the IEEE 802.3 committee — the working group that defines wired network standards to this day.

The committee defines standards that share the same basic frame type and access method. Here's a selection of its timeline:

StandardDescriptionYear
802.3i10 Mbps Ethernet using twisted pair cabling1990
802.3abGigabit Ethernet over twisted pair1999
802.3by25 Gigabit Ethernet over fiber2016
802.3cm400 Gigabit Ethernet over multimode fiber2020
802.3cu100G and 400G Ethernet over single mode fiber using 100 Gbps lanes2021

How Ethernet Transmits Data

Ethernet uses data frames containing MAC addresses to identify computers, and CSMA/CD to determine which machine accesses the wire at any given time.

Using frames solves two critical problems: it prevents a single machine from monopolizing the network, and makes retransmitting lost data more efficient.

// Frame Structure

Building on the frame content from our OSI Model articles:

  • Preamble — a 7-byte series of alternating ones and zeroes followed by a 1-byte start frame delimiter. Sent before the frame to notify the receiving NIC that a frame is coming so it knows where it starts
  • Type field — helps the receiving machine interpret data at a basic level — tells it whether the data is IPv4 or IPv6
  • Pad — extra data added by the sending NIC to bring the frame up to the minimum size of 64 bytes. Used all the time

Early Ethernet — From Bus to Hybrid Star-Bus

Early networks used a true-bus topology — every computer connected to the same cable. After 1990, networks moved to a hybrid-bus topology — a hub at the center repeating data to every connected device except the sender.

// 10BASE-T (1990)

The naming convention: 10 = speed in Mbps, BASE = baseband signal, T = twisted pair.

// Cabling & Connectors

  • Required Cat 3 or higher, 2-pair UTP — one pair to send, one to receive. Techs installed 4 pairs to be future-proof
  • Uses RJ-45 connectors — 8 pins, each connected to a specific wire using color coding
  • Two color coding standards: TIA/EIA 568A and TIA/EIA 568B
// CrimpingInstalling an RJ-45 connector onto a cable is called a crimp. The tool used is a crimper.

// Duplex

  • Half-duplex (older NICs) — either send or receive at a time, not both
  • Full-duplex (modern NICs) — send and receive simultaneously

CSMA/CD — Collision Detection

On bus and hybrid star-bus networks, CSMA/CD (Carrier Multiple Access with Collision Detection) determines which computer uses the shared cable at a given time.

01Machines have equal access to the wire — assigned on a first-come basis
02Two machines simultaneously detect the wire is free and both send a frame — collision occurs, both frames lost
03NICs sense the overlapping signals and both stop transmitting
04Each generates a short random waiting time — shorter wait sends first, the other waits for the wire to be free
// Modern SolutionBetter technologies today — specifically switches — eliminate collisions entirely.

Ethernet Switches

Ethernet switches look like hubs but work completely differently. Think of a switch like a phone operator — handling every connection between two devices as if it's the only connection on the network, running at full speed.

1Initially forwards all messages to every computer (like a hub)
2Registers every MAC address into a MAC table
3Starts filtering messages — creating point-to-point connections between specific devices
  • Each port acts as its own collision domain — two nodes can send data simultaneously without collision
  • Switches can buffer frames — store them briefly if the destination port is busy

// Uplink Ports & Crossover Cables

To avoid single points of failure and expand the network, switches have uplink ports for connecting to other switches. Crossover cables can connect switches or two PCs directly without a switch. In modern switches, any normal port works without special cabling.

Switching Loops & STP

Switching loops occur when two switches are connected in a way that creates redundant connections — causing frames to circulate endlessly and flood the network.

// STP — Spanning Tree Protocol

  • Switches send BPDUs (Bridge Protocol Data Units) to detect potential loops before they happen
  • One switch acts as the root bridge — the center reference point
  • Other switches maintain a loop-free topology based on the root bridge
  • If a link goes down, STP sends a TCN (Topology Change Notification) — switches rework themselves around the failed interface
// RSTPSTP was replaced by the faster and more reliable Rapid Spanning Tree Protocol (RSTP).

// SOC Analyst Relevance

Ethernet basics may seem like pure networking territory, but a few concepts here directly connect to SOC work.

The most practical one is switching loops and broadcast storms. When a switching loop occurs — whether from misconfiguration or a deliberate attack — it generates a massive flood of broadcast traffic that can overwhelm the network. In your SIEM, this shows up as a sudden spike in traffic that looks remarkably similar to a DoS attack. Knowing the difference between a broadcast storm and an actual attack saves you from a false positive investigation.

Understanding how switches build and maintain their MAC tables also remains relevant — as covered in our OSI Model article, MAC flooding exploits exactly this mechanism, forcing a switch to behave like a hub and exposing traffic to all ports.

Finally, STP manipulation is worth being aware of as an advanced attack vector. An attacker sending rogue BPDU frames can force themselves into the root bridge position, intercepting traffic across the network. You won't encounter this daily as a Tier 1 analyst, but recognizing it in a threat intelligence report or an incident escalated from Tier 2 requires knowing what STP is and how it works.

As always — the deeper your understanding of how the network operates normally, the faster you recognize when something is wrong.

// Key Takeaways

Ethernet was standardized by IEEE 802.3 — the committee that still defines wired networking today
Frames use preamble, type field, and pad to structure data transmission reliably
CSMA/CD managed collisions on early networks — switches eliminated them entirely
Switches build MAC tables to create direct point-to-point connections between devices
STP/RSTP prevents switching loops from flooding the network with broadcast traffic
Broadcast storms and STP manipulation are real SOC-relevant threats hiding in Ethernet fundamentals
