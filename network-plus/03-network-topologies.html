What Is a Network Topology?

Topologies are diagrams of how devices are connected. Every network — including wireless ones — implements a topology, just in different forms. A network technology is the practical application of a topology combined with other critical tools — it provides the method to get data from one computer to another.

Old (Legacy) Topologies

Mostly obsolete today but important to understand historically and conceptually.

Bus
// Obsolete

All devices connected with a single wire in a straight line. Data sent on the whole bus — every device sees every transmission. Required termination on both sides.

Cable break reflects data — entire network goes down.
Ring
// Obsolete

Data moves in a circle with no end to the cable — no termination needed.

Cable break = data gets lost, network fails.
Star (Hub & Spoke)
// Obsolete

Devices connected via a Hub as a central box. Offered fault tolerance — one cable breaking didn't take down the whole network.

Hard to redesign existing networks and more expensive.
Hybrid
// Dominant today

Combines physical topology with logical topology. Star-Bus reigns as the undisputed standard of wired topologies. Star-Ring lost market share.

Mesh
// Wireless networks

Each device connects to every other via one or more routes. Partially meshed — at least two devices have redundant connections. Fully meshed — every device connects to every other directly.

Point-to-Point
// Simplest

Connects two network nodes directly. The simplest topology possible.

Logical Topologies

Logical topologies describe how data flows through a network regardless of physical layout.

// 3-Tier Hierarchical Model

The most common enterprise network model:

C
Core Layer
High-speed backbone, internet connection
D
Distribution Layer
Routing, policies, filtering
A
Access Layer
End devices — PCs, printers, phones

// Collapsed Core Model

Core and distribution layers collapsed into a single device. Used in smaller networks — more cost effective but less scalable.

// Spine and Leaf

Modern data center topology. Reduces latency and hop count. Traffic flows east to west only — every leaf connects to every spine, no spine-to-spine or leaf-to-leaf connections. Highly predictable and scalable.

Network Cabling

All network cables fall into two main categories:

⚡ Copper Cabling
  • Coaxial copper
  • Twinaxial cable
  • Twisted pair (STP/UTP)
💡 Fiber Optic
  • Multimode fiber (MMF)
  • Single-mode fiber (SMF)

// Coaxial Copper

Center wire and braided metal shield share a common axis — hence "coaxial". The shield protects data from EMI (electromagnetic interference). Has a Radio Guide (RG) rating for characteristic impedance. Uses BNC connectors and F-type connectors for cable modems.

RatingOhmsUseConnector
RG-5975Cable TVF Type
RG-675Cable TV / ISPF Type

// Twinaxial Cable

Two central copper conductors in a single shield. Used as a short-distance substitute for fiber between rack equipment — called a DAC (Direct Attached Cable).

// Twisted Pair Cable

The most common cabling type in modern networks. Pairs work as a team — transmitting and receiving. Twisting reduces crosstalk.

STP — Shielded Twisted Pairs

NameDescription
F/UTPFoil shields entire cable; inside wires are like UTP
S/UTPBraid screen shields entire cable; inside wires are like UTP
SF/UTPBraid screen and foil shield entire cable; inside wires are like UTP
S/FTPBraid screen shields entire cable; foil shields each wire pair inside
F/FTPFoil screen shields entire cable; foil shields each wire pair inside
U/FTPNo overall shielding; each pair inside shielded with foil screens

UTP — Unshielded Twisted Pairs

Twisted pairs surrounded by a plastic jacket that doesn't prevent EMI — acts as a slight skin only.

// Fiber Optic Cabling

Transmits light rather than electricity — suitable for long distances (10s of km) and high-EMI environments. Structure: glass fiber core → cladding → buffer → insulating jacket. Two fibers connected = duplex fiber-optic cabling (one send, one receive).

TypeLight SourceWavelengthUse

// Modal DistortionMMF uses LEDs and suffers from modal distortion. SMF uses laser light and prevents this — enabling phenomenally higher speeds over longer distances.

// SOC Analyst Relevance

While memorizing every cable specification isn't required, having a working knowledge of network infrastructure is genuinely valuable in cybersecurity.

On the offensive side, red teamers performing physical security assessments often rely on this knowledge — from crimping custom cables to planting rogue devices on a target network. On the defensive side, as the saying goes: you can't defend what you don't understand.

As a SOC analyst working in an organization with a local network, understanding its physical topology helps you detect anomalies and identify unauthorized devices. Knowing the IP ranges, subnets, and expected devices on a network makes anything unexpected stand out immediately.

Physical layer issues can also trigger security alerts in unexpected ways — a broken cable can cause ARP floods or a switch going offline, which may look like a service degradation attack if you don't understand the underlying infrastructure.

Topology knowledge also directly aids incident response. Knowing whether a network uses a flat, star, or hierarchical model tells you instantly how traffic flows and where to look during an investigation. A segmented network behaves very differently from a flat one when you're tracing lateral movement.

While network redesign falls under security engineering, a SOC analyst who understands topology can flag segmentation weaknesses during investigations — for example noting that a compromised device has unrestricted access to critical systems due to a flat network structure. That observation goes into the incident report and becomes actionable for the right team.

Ultimately, even if you don't memorize every spec, keep these as references. The real value comes from hands-on experience — building your own lab, configuring switches, understanding your network's topology. That's where knowledge becomes intuition.

// Key Takeaways

Every network implements a topology — physical or logical or both
Star-Bus hybrid is the dominant wired topology today
Enterprise networks use a 3-tier hierarchical model — Core, Distribution, Access
Copper cables are common and affordable — fiber is faster and EMI-immune over long distances
Cat 5e and above are the current standard for twisted pair cabling
Topology knowledge helps SOC analysts investigate incidents and flag infrastructure weaknesses
