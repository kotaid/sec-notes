What Is a Router?

A router is any piece of hardware or software that forwards packets based on their destination IP address. The typical home router combines multiple functions into one box — router, switch, Wi-Fi WAP, firewall, and DHCP server.

When a packet arrives, the router strips off any L2 information and queues the resulting IP packet. The routing table then tells it exactly where to send it.

The Routing Table

Each row in a routing table defines a single route. The router uses a combination of the destination and subnet mask to match a packet to a route — if multiple routes match, it picks the best one.

NAT — Network Address Translation

To protect LAN computers and conserve public IP addresses, routers use NAT — replacing the source IP address of outgoing packets with the router's public IP.

// PAT — Port Address Translation

The most common form of NAT — handles many-to-one connections using port numbers to map traffic from specific internal machines:

  • When a packet arrives at the NAT device, source/destination IPs and port numbers are recorded in the NAT translation table
  • The private IP is swapped for the public IP on each packet
  • The internal port number is also translated to a unique number
  • Because of the wide TCP/IP port range, PAT can manage thousands of internal connections through a single public address
  • Drawback: PAT works only for outgoing communication

// DNAT — Dynamic NAT

Multiple internal computers share a pool of public addresses — private-to-public mapping happens dynamically from the pool.

// Port Forwarding (SNAT — Static NAT)

For incoming communication, SNAT maps a single routable IP to a single internal machine on a permanent one-to-one basis. Port forwarding designates a specific local address for various network services — external computers access a service using the router's public IP and the service's port number.

// Security NotePort forwarding exposes internal services to the internet. Every open forwarded port is an attack surface — always restrict to the minimum necessary and ensure the service is hardened.

Dynamic Routing Protocols

Most routers can update their routes dynamically — routing protocols allow routers to talk to each other, share topology information, and adapt when routes go down.

// Metric Criteria

Hop Count

Number of intermediate routers between source and destination

Bandwidth

Some connections handle more data than others

Delay

Latency issues that prevent or slow down connections

Cost

Low-bandwidth connections have higher cost — admins can add cost to routes

// Routing Protocol Types

RIPv1
Distance Vector
  • Max 15 hop count
  • Updates every 30 seconds — caused network overload
  • No VLSM support, no authentication
  • Largely obsolete
RIPv2
Distance Vector
  • Supports VLSM and authentication
  • Multicast instead of broadcast
  • Updates every 90 seconds
  • Slow convergence — suitable for simple networks only
OSPF
Link State — IGP
  • Most used interior gateway protocol in the world
  • Designed for single AS environments
  • Uses Hello packets for neighbor discovery
  • Exchanges LSAs to build Link-State Database
  • SPF algorithm calculates shortest paths
  • Hierarchical areas for scalability
IS-IS
Link State
  • Originally designed for CLNS, adapted for IP
  • Supports IPv6 natively
  • Not widely used
BGP
Path Vector — EGP
  • The foundational protocol of the internet
  • Operates between autonomous systems (AS)
  • Focuses on routing policy, not physical path
  • Can aggregate multiple routes into one
  • Considered hybrid — combines distance-vector and path-vector
EIGRP
Hybrid — Cisco Proprietary
  • Cisco's enhanced interior gateway protocol
  • Combines aspects of distance vector and link state
  • Fast convergence
  • Cisco devices only
// Route RedistributionWhen a router advertises routes learned from one routing protocol into another — allowing networks using different protocols to share routing information and work together. Uses administrative distance to determine the most reliable route.

Router Configuration

New routers require initial configuration — connected via a rollover cable and a terminal emulation program like PuTTY. Most Cisco products run Cisco IOS.

// Configuration Best Practices

  • Always fully configure before deploying — avoid rogue DHCP issues and route overlaps
  • Add strong passwords immediately — change all defaults
  • Secure the web interface — set a static IP first if needed
  • Configure the WAN side — most routers use DHCP to connect to the ISP
  • Choose a private network ID for the LAN side unless NAT is not needed
  • Back up configurations and document all routes
// Static Routes = First SuspectIf data isn't going where it's supposed to, static routes are the main suspects.

// MTU and Fragmentation

Network components split packets or frames to stay below the MTU (Maximum Transmission Unit) threshold for the next hop. Fragmentation increases the number of packets needed — a gross inefficiency.

  • Path MTU — the largest packet size transmissible without fragmentation through all hops
  • Path MTU Discovery — devices determine the path MTU dynamically
  • MTU size can be adjusted at the source to eliminate fragmentation entirely

// Diagnostic Tools

  • Traceroute — records the route between any two hosts on a network
  • My Traceroute (mtr) — dynamic version, continually updates the selected route
  • Pathping — Windows equivalent, performs a similar function

// SOC Analyst Relevance

Routing knowledge sits at an interesting crossroads — it's foundational enough that every security professional benefits from it, yet deep enough that specialists like cloud security engineers and analysts rely on it daily. If you ever find yourself building cloud infrastructure, setting up virtual labs, or analyzing cloud-native security alerts, you'll quickly realize that the core concepts are the same — just implemented differently. NAT, PAT, routing tables, port forwarding — these appear constantly in cloud environments under different names but with the same underlying logic.

Routers are, as we touched on in a previous article, high-value targets. They control the flow of all traffic — internal and external — making any successful attack on them potentially catastrophic. While the specific attack techniques that target routing protocols tend to be advanced and typically occur in large or specialized enterprise environments, the awareness that routers are precious and heavily targeted assets should shape how you think about network security as a whole.

From a practical standpoint, you cannot build a meaningful home lab — local or cloud-based — without understanding routing. Virtually every network appliance including routers is available in virtualized form, and the concepts translate directly. Knowing how to read a routing table, understand NAT, or configure basic static routes is what separates someone who can build a functional lab environment from someone who can't.

On the hardening side — changing default passwords, securing the web interface, and fully configuring a router before deployment are things a SOC analyst would flag during a security audit or when reviewing an incident report involving a compromised network device. You may not be the one doing the configuring, but recognizing a misconfigured or default-credential router as a security gap is absolutely within your scope.

And as always — you don't need to memorize every routing protocol, every metric type, or every flag in a routing table. Having heard the terminology, knowing that things work in a particular way, and being able to recognize them when they appear in logs or documentation is what matters. That recognition is what makes you a well-rounded analyst.

// Key Takeaways

Routers forward packets based on destination IP — stripping L2 info and consulting the routing table
Every device — not just routers — has a routing table defining how to handle different traffic types
NAT/PAT allows many internal hosts to share a single public IP — fundamental to modern networks
Dynamic routing protocols let routers adapt automatically when network topology changes
OSPF is the dominant interior protocol; BGP is what holds the internet together
Always fully configure and harden a router before deploying — defaults are a security liability
